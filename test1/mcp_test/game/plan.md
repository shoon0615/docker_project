# Vampire Survivors-like Game Development Plan

This document outlines the development plan for a 2D auto-shooter game inspired by Vampire Survivors.

## Phase 1: Basic Game Setup (Completed)

*   [x] Create project directory: `mcp_test/game`
*   [x] Create `index.html` for the basic game structure.
*   [x] Create `style.css` for basic styling.
*   [x] Create `game.js` for the game logic.
*   [x] Implement basic game loop.
*   [x] Implement player character with basic movement (WASD/Arrow keys).
*   [x] Implement basic automatic shooting mechanism.
*   [x] Implement basic enemy spawning.
*   [x] Implement basic collision detection (player-enemy, bullet-enemy).
*   [x] Implement basic scoring system.
*   [x] Implement game over condition.

## Phase 2: Core Gameplay Enhancements

*   [ ] **Weapon System:**
    *   [ ] Implement different weapon types (e.g., projectiles, area of effect).
    *   [ ] Add weapon upgrade system.
    *   [ ] Allow player to choose from a selection of weapons/upgrades upon leveling up.
*   [ ] **Experience and Leveling:**
    *   [ ] Enemies drop experience gems upon death.
    *   [ ] Player collects gems to gain experience.
    *   [ ] Player levels up upon reaching experience thresholds.
*   [ ] **Enemy Variety:**
    *   [ ] Introduce different enemy types with varying health, speed, and behavior.
    *   [ ] Implement enemy waves with increasing difficulty.
    *   [ ] Add boss encounters.

## Phase 3: Graphics and Audio

*   [ ] **Sprites:**
    *   [ ] Replace primitive shapes with custom sprites for player, enemies, and projectiles.
    *   [ ] Add animations for player movement, attacks, and enemy deaths.
*   [ ] **Visual Effects:**
    *   [ ] Add particle effects for impacts, explosions, and level-ups.
    *   [ ] Implement a scrolling background to give a sense of movement.
*   [ ] **Audio:**
    *   [ ] Add background music.
    *   [ ] Add sound effects for shooting, enemy hits, and player death.

## Phase 4: UI/UX Improvements

*   [ ] **Heads-Up Display (HUD):**
    *   [ ] Display player health, experience bar, level, and current weapon.
    *   [ ] Show a timer for game duration.
*   [ ] **Menus:**
    *   [ ] Create a start menu.
    *   [ ] Implement a pause menu.
    *   [ ] Add a game over screen with stats (score, time survived, etc.).
*   [ ] **Controls:**
    *   [ ] Add support for gamepads.

## Phase 5: Polishing and Deployment

*   [ ] **Balancing:**
    *   [ ] Fine-tune game difficulty, weapon power, and enemy stats.
*   [ ] **Bug Fixing:**
    *   [ ] Thoroughly test the game and fix any bugs.
*   [ ] **Deployment:**
